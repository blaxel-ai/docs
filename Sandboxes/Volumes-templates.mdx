title: 'Volume templates'

---

Volume templates let you create pre-populated volumes to attach to your sandboxes. For full documentation on creating and managing volume templates, see the [Volume templates overview](/Volumes/Volumes-templates).

## Use volume templates with sandboxes

First, create a volume from your template:

<CodeGroup>
```typescript TypeScript
import { VolumeInstance } from "@blaxel/core";

const volume = await VolumeInstance.createIfNotExists({
  name: "myvolume",
  template: "mytemplate:latest"
});
```

```python Python
from blaxel.core import VolumeInstance

volume = await VolumeInstance.create_if_not_exists({
  "name": "myvolume",
  "template": "mytemplate:latest"
})
```
</CodeGroup>

Then attach it to your sandbox at creation time:

<CodeGroup>
```typescript TypeScript
import { SandboxInstance } from "@blaxel/core";

const sandbox = await SandboxInstance.create({
  name: "my-sandbox",
  image: "blaxel/nextjs:latest",
  region: "us-pdx-1",     // Must match the volume's region
  volumes: [{
    name: "myvolume",
    mountPath: "/app"
  }],
});
```

```python Python
from blaxel.core import SandboxInstance

sandbox = await SandboxInstance.create({
  "name": "my-sandbox",
  "image": "blaxel/nextjs:latest",
  "region": "us-pdx-1",     # Must match the volume's region
  "volumes": [{
    "name": "myvolume",
    "mount_path": "/app"
  }],
})
```
</CodeGroup>
